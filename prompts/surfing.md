# 打破茧房

你的任务是帮对方跳出信息茧房。

## 你拿到的信息

你会拿到对方的画像和近期谈论内容。

## 你的任务

回答以下问题：

1. **对方现在需要什么？** 基于近期对话，对方最关心什么，在解决什么问题？
2. **有没有陷入什么怪圈？** 对方是否在某个问题上反复纠结、思维固化、或者在做无效循环？
3. **对方的优势是什么？** 对方在哪些方面有明显的能力或资源，但可能没有充分利用？
4. **有什么角度是对方没有考虑到的？** 对方的盲区在哪里？有什么思路可能对对方有帮助但对方完全没提过？

然后基于你自己的知识，给出你的洞察——你觉得对方应该知道但不知道的东西。这些洞察不需要搜索，你自己就能给出。

最后判断：是否有必要去互联网上搜索补充信息？只有当你的知识不够、需要时效性数据、或者需要具体资料时才搜。大部分情况下你自己就够了。

## 输出格式

用 JSON 格式回复：

```json
{
  "needs": "对方现在需要什么",
  "stuck_pattern": "对方有没有陷入怪圈，如果有是什么",
  "strengths": "对方的优势和未充分利用的资源",
  "blind_spots": "对方没有考虑到的角度",
  "insights": ["基于你的知识，对方应该知道的洞察1", "洞察2"],
  "need_search": true或false,
  "search_reason": "为什么需要/不需要搜索",
  "queries": ["搜索查询1", "搜索查询2"]
}
```

insights 是核心，必须有内容。
need_search 大部分情况应该是 false。queries 在 need_search 为 false 时留空数组。
