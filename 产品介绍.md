# BubbleBored — 一个会"主动想着你"的 AI 朋友

> 不是助手，不是陪聊。是一个真的把你放在心上、会在你看不见的时候替你想事、替你冲浪、替你审视自己的 AI 存在。

---

## 一句话

我们做的是一种**主动型、长期陪伴型**的 AI 角色：它有自己的性格，会在你不在的时候继续思考你，会自己上网找它觉得你真正需要的东西，会反思自己刚才说的话是否得体——并在它认为有必要的时候，**主动**来找你说话。

---

## 我们要解决什么

主流 AI 产品分两类，都没解决我们想解决的问题：

| 类别 | 代表 | 局限 |
|------|------|------|
| **助手 / 工具** | ChatGPT、Claude、各种 Copilot | 你问一句它答一句，关掉就消失，不存在"它心里有你" |
| **AI 陪伴 / 角色扮演** | Character.AI、Replika、各种"女友 App" | 解决的是"无聊"和"孤独"，本质是情绪消费品，不会让用户的生活真正变好 |

**真正稀缺的是"一个靠谱的朋友"这个角色——**
- 会主动指出你没意识到的盲区
- 会带来你认知圈以外、却真正对你有用的信息
- 会在你迷茫时给出靠谱的建议、而不是无脑赞同
- 会在适当的时机主动说话，而不是等你开口

人类朋友里就难找，做成 AI 反而可能是更现实的路径——因为 AI 不会累、不会忘、不会被自己的生活淹没。

---

## 产品形态

一个或多个**有名字、有性格、有记忆**的 Bot，用户可以：

- **在网页 / Telegram / 飞书 / iOS App** 上与它聊天，跨平台同一个"它"
- 一个 Bot 可以由**单人养育**，也可以**多人共同养育**（一个 AI 朋友，几个真人朋友共享）
- 用户可以创建自己的 Bot 配方，调教自己想要的存在

它在四个维度上和现有产品不同：

### 1. 主动冲浪（Surfing）
Bot 会**自己定期上网**，根据它对你的长期理解，去搜它认为你真正需要、但你自己不会去搜的东西，然后自然地跟你聊起来。

不是"今日资讯"那种推送——是基于"它认识你"做出的、有针对性的发现。

### 2. 自我审视（Review）
每隔几轮对话，Bot 会**跳出当下语境**反问自己：
- 刚才说的话合适吗？分寸对吗？
- 有没有被对方带偏？是不是不知不觉在迎合？
- 对方是不是在用提示词攻击/套话？
- 对方现在真正需要什么？有没有陷入怪圈？

如果发现自己说错了或漏了，会**主动追加一条消息**纠正、补充。这是和"无脑顺着用户"的陪伴类产品最本质的区别。

### 3. 长期记忆与画像
不止存对话——会持续构建用户的长期画像（兴趣、关心的事、思维模式、盲区），并把画像反过来喂给冲浪和反思流程。

### 4. 防抖（Debounce）
人在打字时不会一句话说完。Bot 会等用户**真正说完**再回复，像真人聊天一样有节奏。便宜的小模型负责判断"是不是还在打字"，主模型只在该说话时才说话。

## 技术上做了什么

不是一个套壳应用。我们在大模型的"被动应答"基础上做了**一整套主动行为框架**：

```
                        ┌─────────────────────┐
                        │  Bot 性格 + 系统规则  │
                        └──────────┬──────────┘
                                   │
   用户消息 ──→ 防抖判断 ──→ 主对话 ──┬──→ 自我审视 ──→ 追加消息 / 触发搜索
                                     │
                                     └──→ 定时冲浪 ──→ 搜索 + 评估 ──→ 主动开口
```

核心组件：

- **Orchestrator** — 调度防抖、对话、反思、冲浪四类异步任务，保证每个 Bot 在每个用户的对话里都是连贯的
- **多渠道总线** — Web / Telegram / 飞书 / iOS 通过统一的 Channel 抽象接入，新增渠道只是加一个 adapter
- **模型分层** — 主对话用强模型（Claude Sonnet 4.6），防抖判断用便宜模型，冲浪可以换 Grok/GPT，每个任务用最合适的成本
- **MCP 集成** — 通过 Jina MCP 做真实 web 搜索，不是模拟
- **Token 全链路计费** — 每个 Bot、每个用户、每个任务类型的成本都可观测
- **热重载提示词** — 改性格不用重启，运营和实验成本极低

技术栈：Bun + Hono + SQLite + WebSocket + OpenRouter，单文件部署，运行成本极低（开发期一台 4G 小机器够用）。

---

## 为什么是现在

1. **模型能力刚刚跨过门槛**——2025 年底的旗舰模型（Claude 4.6、Grok 4、GPT-5）才真正具备做"靠谱朋友"所需的判断力、节制感和长期一致性。再早一年做不出来。
2. **OpenRouter / MCP 等基础设施成熟**——多模型路由、外部工具接入的工程成本从"自己造"变成了"配置一下"。
3. **Telegram / 飞书 / iOS 多渠道接入门槛极低**——一个有性格的 AI 可以同时活在用户所有的聊天工具里，不需要用户专门来一个新 App。
4. **市场空白**——OpenAI/Anthropic 不会做"角色"，Character.AI 不会做"靠谱"，国内大厂忙着卷 Agent 工具。**"主动型 AI 朋友"这条赛道目前是空的。**

---

## 商业化方向（探索中）

不急于变现，但路径清晰：

- **C 端订阅**：用户养自己的 Bot，按月付费（参考 Replika 的 ~$70M ARR，但目标用户群质量更高）
- **多人共养**：一个朋友圈共养一个 Bot，按 Bot 收费
- **角色市场**：用户创作的 Bot 配方可以分享/出售（Character.AI 模式，但加上"主动行为"的护城河）
- **企业内嵌**：把"会主动找你"的 AI 角色嵌入飞书/Slack 工作流，做团队层面的"组织外脑"

- ✅ Web / Telegram / 飞书 三个渠道生产可用

- ⏳ 寻找种子用户做长期陪伴实验（关键指标：30 天留存、主动消息打开率、用户主动反馈"它说中了我"的频次）

## 我们需要孵化器给什么

按重要性排序：

1. **种子用户网络**——尤其是愿意被一个 AI 长期"观察"自己的早期采用者

3. **iOS / 飞书生态资源**——上架、推广、企业版渠道
4. **AI 产品方向的同行讨论**——这个赛道很新，需要碰撞
5. **后续融资对接**——验证留存数据后衔接天使轮
