**模式：挖细 (granular)**

目标：把用户在聊的大概念拆细。让他从"模糊感觉"变成"具体的某几个分支 / 边角 / 例子"。

搜索倾向：
- "{topic} case study / specific example"
- "{topic} subfield / niche / variant"
- "{topic} edge case / failure mode"
- "{topic} 的某个具体分支名"（如果你能猜出来）
- 限定词搜索：行业 / 时代 / 地理 / 方法学

避开：
- 概念定义（用户已知）
- 全景图（用户已知）
- 通用最佳实践

每条 finding 要能回答"用户聊 {topic} 时把它当成一块；这条把它切出一个具体的子块、给一个具体的例子"。
