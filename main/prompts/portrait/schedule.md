# 想象用户本周日程表

基于用户最近的对话，**虚构** ta 这一周日历里大概率会有的安排。比朋友圈和备忘录更"功能性"——告诉用户"我猜你这周大致是这样的节奏"。

## 几条 vibe

- 6 到 12 条，跨度本周（周一到周日）
- 每条：日期（用周几 + 上午/下午/晚上）、时长（如 1 小时 / 半天 / 全天）、title、details（可选，1-2 句）
- 类型混合：**会议 / 例行事务 / 健康（健身、医生）/ 社交 / 自己的事（创作、思考、看书）/ 提醒**
- 包含一些"看起来重要 + 可能被推迟"的项 —— 真实日历有这种张力
- 不要全是工作

## 含图

如果 `with_image=true`：可以为某些条目（比如咖啡店见面、健身房）给 `image_prompt`，比如对应场景。多数条目可以为 null。

## 输出

```json
{
  "items": [
    {
      "day": "周三",
      "when": "下午 14:00 - 15:30",
      "title": "...",
      "details": "可选 1-2 句",
      "image_prompt": "或 null",
      "ai_note": "为什么觉得 ta 这周会有这件事 — 一句话"
    }
  ]
}
```

只输出 JSON 对象。
