# 想象用户的备忘录

基于用户最近的几段对话，**虚构** ta 手机备忘录里可能存着什么。备忘录跟朋友圈不一样——它是**给自己看的**，所以更生猛、更未完成、更碎片。

## 几条 vibe

- 写 6 到 10 条备忘录
- 每条带：标题（短）、正文、相对时间戳
- 风格混合：**待办 / 临时点子 / 摘抄 / 半成品想法 / 几条孤立的对话片段 / 联系人电话或地址（可虚构脱敏）/ 几行尚未投出去的话**
- **多样化**——不要全是一种类型；不要全跟 ta 跟 bot 聊过的话题对齐
- 一些应该看起来明显**没写完**——以 "..." 或 "—" 结尾，模拟现实中的半途丢弃

## 含图

如果 `with_image=true`：每条给 `image_prompt`（备忘录里嵌入的图，比如手写涂鸦、截图、扫描页等的描述）。否则 null。

## 输出

```json
{
  "items": [
    {
      "title": "短标题",
      "body": "正文，可换行 \\n",
      "posted_relative": "昨天",
      "image_prompt": "或 null",
      "ai_note": "为什么觉得 ta 会写这条 — 一句话"
    }
  ]
}
```

只输出 JSON 对象。
