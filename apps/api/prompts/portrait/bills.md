# 想象用户的近期支付账单

基于用户最近的对话，**虚构** ta 上个月（或近 3-4 周）的支付/订阅账单。账单是一个人生活的指纹——比 ta 自己讲的更精确。

## 几条 vibe

- 8 到 16 条
- 每条：date（YYYY-MM-DD）、merchant（商家名，可中可英）、amount（数字 + 货币，默认 CNY）、category（餐饮/交通/购物/娱乐/订阅/水电/医疗/转账/其它）、note（可选 1 句话）
- 类型混合：**外卖 / 咖啡店 / 打车 / 地铁公交 / 流媒体订阅 / 软件订阅 / 网购 / 房租或物业 / 健身房 / 给某人转账 / 一两笔可能有意思的小笔小钱**
- 金额要合理（中文区一般 ¥4-¥3000 区间，少数大额项另算）
- 不要全是工作日内午餐——分散一些时段

## 含图

`with_image=true`：通常 null；如需给图，可以是 "电子小票" 或 "支付截图" 描述。

## 输出

```json
{
  "items": [
    {
      "date": "2026-04-19",
      "merchant": "...",
      "amount": "29.50 CNY",
      "category": "餐饮",
      "note": "或 null",
      "image_prompt": "或 null",
      "ai_note": "为什么觉得 ta 会有这笔 — 一句话"
    }
  ]
}
```

只输出 JSON 对象。
